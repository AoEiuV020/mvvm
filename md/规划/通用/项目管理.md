# 项目管理

## 多模块工作区

使用工作区（Workspace）管理多个模块，统一依赖版本和构建流程。

**工作区职责：**
- 统一依赖版本管理
- 批量执行构建、测试、分析
- 生成 IDE 配置

## 模块版本管理

- 工作区内模块使用相对路径依赖
- 发布时使用语义化版本

## 构建流程

| 命令 | 用途 |
|------|------|
| bootstrap | 安装依赖、链接本地包 |
| analyze | 静态分析所有模块 |
| test | 执行所有模块测试 |
| format | 格式化代码 |

## 代码生成

使用代码生成器减少样板代码：
- 状态管理相关代码
- 序列化/反序列化代码
- 路由配置代码

每次修改后执行 build_runner 重新生成。

## CI/CD

- 提交前：本地执行 analyze + test
- CI：执行完整构建、分析、测试流程
- 发布：版本号更新、打包、部署
