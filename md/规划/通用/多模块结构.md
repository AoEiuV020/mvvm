# 多模块结构约束

## 模块划分

| 模块 | 职责 | 依赖 |
|------|------|------|
| `sdk` | 核心 SDK，无关 UI | 无 UI 框架依赖 |
| `sdkui` | UI 层 SDK（ViewModel + 可复用组件） | sdk |
| `uikit` | 纯 UI 组件库（无业务） | 无业务依赖 |
| `app` | 应用入口，组装页面 | sdkui |

## 分层关系

```
app → sdkui → sdk
        ↓
      uikit
```

## 模块职责详情

### sdk
- 接口定义：IAuthService, IMessageService 等
- 数据模型：User, Message, Conversation 等
- 接口实现

### sdkui
- ViewModel 接口和实现
- 状态类定义
- 可复用业务组件（消息气泡、会话列表项等）
- 依赖注入配置

### uikit
- 纯 UI 组件（按钮、输入框、头像等）
- 主题定义
- 无业务逻辑

### app
- SDK 初始化
- 路由配置
- 页面组装
- 应用定制

## 命名规则

具体模块名由各框架约束文档定义（如 Flutter 使用下划线命名）。
