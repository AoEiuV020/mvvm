# 规划文档约束

## 框架目录结构规范

每个 UI 框架目录（如 `flutter/`、`arkts/`、`vue3/`）必须包含以下文档：

| 文件 | 职责 |
|------|------|
| `README.md` | 入口文档，包含技术栈和文档索引 |
| `项目结构.md` | 目录结构和文件命名规范 |
| `SDK使用方式.md` | SDK 初始化、Provider/依赖注入配置 |
| `状态管理.md` | ViewModel 实现模式、状态更新规则 |
| `复杂页面组织.md` | 多 ViewModel 拆分、View 组合、通信 |
| `路由导航.md` | 路由配置和导航方法 |
| `测试规范.md` | 单元测试和 UI 测试模式 |
| `跨页面状态共享.md` | 多页面共享 ViewModel 实例的模式 |

## 文档内容规范

1. **README.md**
   - 列出技术栈（库名 + 用途）
   - 提供文档索引表格
   - 说明 MVVM 在该框架下的对应关系

2. **其他文档**
   - 只包含代码模式和示例
   - 示例使用 `Xxx` 占位符，不写具体业务代码
   - 保持精简，避免重复

## 禁止内容

- 版本号
- 依赖配置文件模板（pubspec.yaml、package.json）
- 安装命令
- 冗长的说明文字
