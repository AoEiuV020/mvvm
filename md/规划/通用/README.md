## 文档规范

每个功能文档应包含：
- 功能描述
- UI 布局说明
- Model 层：依赖的 SDK 接口
- ViewModel 层：接口定义、状态说明、行为逻辑
- View 层：职责和用户交互

**语言要求：**
- 使用框架无关的语言描述
- ViewModel 接口使用 Dart 伪代码定义
- 不得出现具体框架的实现方式（如 Riverpod、Pinia）
- 具体实现方式由各框架文档定义
